# Support

## 🆘 Getting Help

### Quick Fixes
- **No sound?** Check the [Troubleshooting section](README.md#troubleshooting) in README
- **Extension not working?** Try restarting VS Code (`Ctrl+Shift+P` → "Developer: Reload Window")
- **Settings not saving?** Check that VS Code has permission to write settings

- [README.md](README.md) - Full documentation

### Debug Information
Enable debug mode to see detailed logs:
1. Press `Ctrl+Shift+P`
2. Type "FAAH: Toggle Debug"
3. Check the Output Panel (`Ctrl+Shift+U` → Select "FAAH Terminal Alert")

## 🔄 Updates

### How to Update
The extension updates automatically through VS Code. To check for updates:
1. Go to Extensions (`Ctrl+Shift+X`)
2. Search for "FAAH Terminal Alert"
3. Click "Update" if available

##  FAQs

**Q: Can I customize the sound?**
A: Currently not supported, but this is planned for a future release.

**Q: Does it work with integrated terminals only?**
A: Yes, it monitors VS Code's integrated terminals.

**Q: Can I set different sounds for different error types?**
A: Not yet, but we're considering this for v2.0.

**Q: Does it work with remote development?**
A: Audio playback may not work over SSH due to lack of audio device.
